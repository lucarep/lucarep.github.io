WHAT IS AN ONLINE ALGORITHM?⌗

Citing Wikipedia:

In computer science, an online algorithm is one that can process its input piece-by-piece in a serial fashion, i.e., in the order that the input is fed to the algorithm, without having the entire input available from the start. In contrast, an offline algorithm is given the whole problem data from the beginning and is required to output an answer which solves the problem at hand. (For example, selection sort requires that the entire list be given before it can sort it, while insertion sort doesn’t.)

We can then summarize this definition by saying: an online algorithm takes as input a stream of continuous data, while an offline algorithm attempts to solve the problem by already having all the data available from the first moment.

KAHAN SUMMATION ALGORITHM:⌗

A code version of the algorithm, is the following one introduced by KAHAN, which is designed to minimize errors, this is done by keeping a separate running compensation (a variable that accumulates small errors).

function KahanSum(input)
    var sum = 0.0
    var c = 0.0                 // A running compensation for lost low-order bits.
    for i = 1 to input.length do
        var y = input[i] - c    // So far, so good: c is zero.
        var t = sum + y         // Alas, sum is big, y small, so low-order digits of y are lost.
        c = (t - sum) - y       // (t - sum) cancels the high-order part of y; subtracting y recovers negative (low part of y)
        sum = t                 // Algebraically, c should always be zero. Beware overly-aggressive optimizing compilers!
    next i                      // Next time around, the lost low part will be added to y in a fresh attempt.
    return sum

WHY IS AN ONLINE ALGO BETTER IN THIS CASE?⌗

the answer to the question starts with one word digitalization. We live in an increasingly connected world, where more and more applications are entrusted to computers, so it is unthinkable to be able to store a large amount of data in RAM or hard drive and then use them later, because they take up a lot of space and resources. That’s why we need something more efficient, so over the years the online approach has gained more and more attention, because it allows us to save precious resources. Of course, it’s not all gold that glitters, in fact, for problems that don’t require large amounts of data, an offline approach is preferable because it’s easier to implement and in some cases may even be more efficient.
